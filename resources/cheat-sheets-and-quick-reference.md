## Overview

Cheat sheets are single-page (or single-screen) condensed references designed for active use while building and operating an IRM program — not for reading end-to-end. Each sheet is available as a downloadable PDF and as a Markdown file in the GitHub repository. They are most useful when you are mid-implementation and need a fast lookup without pausing to re-watch a video or search the full handbook.

## Available Cheat Sheets

| Cheat Sheet | What It Covers |
|---|---|
| IRM Terminology Glossary | Definitions for every core IRM term: Risk Statement, Control Objective, Indicator, Finding, Entity, Profile, CRI Profile, POAM, etc. |
| ServiceNow IRM Module Navigation | Where to find each module in the application navigator; keyboard shortcuts; key list views and form views per module |
| Risk Scoring Reference | Likelihood × Impact matrix; score band definitions (Low/Medium/High/Critical); inherent vs. residual risk calculation |

| CRI Profile Tier Map | CRI Profile tier structure, diagnostic statement naming convention, and mapping to NIST CSF and FFIEC CAT |
| KRI Design Checklist | 10-point checklist for designing a usable KRI: measurability, data source, frequency, threshold logic, escalation path |
| Third-Party Risk Tiering Guide | Criteria for tiering vendors as Tier 1 (Critical) / Tier 2 (High) / Tier 3 (Standard); assessment frequency per tier |

## Key concepts

* **Format** — Each cheat sheet is a single-page PDF optimized for A4/Letter printing and a two-page digital view. The Markdown source is also provided so you can embed the content in your internal wikis or adapt it for your organization.
* **When to use them** — Cheat sheets are most valuable during (a) initial implementation when terminology is new, (b) active building sessions alongside a video, and (c) onboarding new GRC team members who need orientation without full handbook depth.
* **Keeping current** — Cheat sheets are version-tagged alongside update sets and diagrams. When a new ServiceNow release changes navigation paths or module names, the relevant cheat sheet is updated and re-released.

{% hint style="info" %}
Print the **IRM Terminology Glossary** and the **Risk Scoring Reference** before your first design workshop with business stakeholders. Aligning on terminology in the first session saves hours of rework later.
{% endhint %}

## Key takeaways

* Cheat sheets are active-use tools, not reading material — keep the relevant sheet open while building.
* The Terminology Glossary is the highest-priority sheet for teams new to ServiceNow IRM — terminology misalignment is the most common source of implementation confusion.
* The KRI Design Checklist prevents the most common KRI configuration mistake: creating indicators that cannot be measured from available data.
* All sheets are available in Markdown for embedding in your organization's internal documentation or wiki.
