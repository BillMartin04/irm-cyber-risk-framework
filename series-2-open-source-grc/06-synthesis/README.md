# Synthesis: The Integrated View

> The framework is not five separate ideas. It is one system, read in sequence.

This section brings the pillars together into a single, coherent thesis. Each layer builds on the one before it:

1. **Risk Universe** — define the whole problem space.
2. **9 Domains** — segment it into ownable areas.
3. **4 Lines of Defense** — assign ownership, oversight, and assurance.
4. **Intelligence (AI)** — augment every domain and line with automation and analytics.
5. **5-Level Maturity Model** — measure where you stand and chart the path forward.

## The core argument

Traditional GRC is fragmented because it starts with tools and works inward. This framework inverts that: it starts from a complete map of risk, structures it, assigns accountability, applies intelligence, and measures maturity — producing an integrated risk practice rather than a collection of disconnected controls.
